# 祖先質量輸送解

この解では、停止前prefixを深さごとの単体としてではなく、各prefixが次のprefixを生む「親子の質量輸送」として数える。

独立な一様乱数`U_1,U_2,...`について

```text
S_0 = 0,
S_n = U_1 + ... + U_n,
tau = min { n >= 1 : S_n > 1 }
```

とする。停止前に見えるprefixを数えれば

```text
tau = sum_{n >= 0} 1_{S_n <= 1}
```

なので

```text
E[tau] = sum_{n >= 0} P(S_n <= 1)
```

である。

## 残り容量ごとの子孫質量

残り容量が`r`のprefixを一つ固定する。このprefix自身は停止前prefixなので、まず質量`1`を持つ。

その子は、次の一口`u`が

```text
0 <= u <= r
```

を満たすときだけ生き残る。子の残り容量は`r-u`である。

残り容量`r`のprefixから下に伸びる全ての子孫prefixの期待Lebesgue質量を

```text
M(r)
```

と書く。親自身の質量と、全ての子から降りる質量を足すと

```text
M(r) = 1 + int_0^r M(r-u) du.
```

変数を替えて

```text
M(r) = 1 + int_0^r M(v) dv.        (1)
```

となる。

ここで使っているのは、各停止前prefixには一つだけ親があり、親から子への許される一口の幅がちょうど残り容量`r`である、という質量輸送である。同じ子孫を二重に数えていない。

## 親子輸送の保存式

(1)を微分すると

```text
M'(r) = M(r),
M(0) = 1.
```

したがって

```text
M(r) = exp(r).
```

初期状態では残り容量が`1`なので、全停止前prefixの期待総量は

```text
M(1) = e.
```

よって

```text
E[tau] = e.
```

この証明では、`n`段目の体積`1/n!`を求めない。停止前prefix全体を、親自身の質量`1`と子への一様な質量輸送に分ける。輸送先の残り容量が再び同じ関数`M`で測られるため、総量が自分自身の原始関数として閉じ、指数関数が現れる。
