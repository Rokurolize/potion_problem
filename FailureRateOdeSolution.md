# failure rate ODE解

この解では、容量が増えたときに期待prefix数がどう増えるかだけを見る。

容量を`x`とした問題の期待停止時刻を

```text
F(x)=E[tau_x]
```

とする。ただし

```text
tau_x=min { n>=1 : U_1+...+U_n>x }.
```

容量を`x`から`x+dx`へ少し増やしたとき、新たに許されるprefixは、和がちょうど`x`付近にあったprefixから生まれる。次の余裕`dx`に入る確率密度は、そこまでのprefix期待密度`F(x)`に等しい。従って

```text
F'(x)=F(x).
```

容量`0`では最初の一回で必ず超えるが、空prefixを尾和に含めるため

```text
F(0)=1.
```

よって

```text
F(x)=e^x.
```

元の問題は`x=1`なので

```text
E[tau]=F(1)=e.
```

同じことを尾和で書けば、`F(x)=sum_{n>=0}P(S_n<=x)`であり、微分すると各密度列が一つ前へずれて`F'(x)=F(x)`になる。
