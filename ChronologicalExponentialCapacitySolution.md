# chronological exponential capacity解

この解では、容量が減る過程を時刻順指数として展開する。

残り容量が`x`のとき、停止前prefix期待数を`V(x)`とする。空prefixを一つ数え、次の入力`u`が容量内に収まると`x-u`へ移るので

```text
V(x)=1+int_0^x V(x-u)du.
```

これは、容量軸に沿って「一つ入力を挿入する」演算を積み重ねる式である。点`0<=t<=x`に入力境界を一つ置く微小演算を`A(t)dt`と書くと、この問題では異なる時刻の演算が可換なので、時刻順指数は普通の指数へ潰れる。

具体的には、`n`個の入力境界を時刻順に置く寄与は

```text
int_{0<=t_1<=...<=t_n<=x} dt_1...dt_n = x^n/n!.
```

従って全prefix期待数は

```text
V(x)=sum_{n>=0} x^n/n! = exp(x).
```

初期容量は`1`だから

```text
E[tau]=V(1)=e.
```

この視点では、停止前prefix列を容量軸上の時刻順配置として読み、時刻順指数の全次数和として`e`が現れる。
