# Lean 4 実装推進プロンプト

## ミッション：媚薬問題の形式的証明を一歩前進させる

### 作業環境
- 作業ディレクトリ: `/home/ubuntu/workbench/projects/potion_problem/lean/`
- Lean 4 with mathlib4 v4.12.0
- 目標: E[τ] = e の完全な形式的証明

### 現在地の確認
1. `lake build` を実行し、現在のビルド状態を確認
2. 各 .lean ファイルの sorry 数とコンパイルエラーを記録
3. 最も取り組みやすい課題を特定

### 今回のタスク（以下から1つ選択）

#### A. Sorry の解決
- 最も単純な sorry を1つ選択
- その sorry が主張する数学的事実を理解
- mathlib の適切な定理を探すか、直接証明を構築
- 証明完了後、`lake build` で検証

#### B. コンパイルエラーの修正
- エラーメッセージを正確に読み取る
- 型の不一致や missing imports を修正
- API の変更に対応（v4.12.0 での制約を考慮）
- 修正後、`lake build` で検証

#### C. 既存証明の改善
- 冗長な証明ステップを特定
- より直接的な証明経路を探索
- mathlib の便利な定理を活用
- 証明の可読性を向上

### 実行ガイドライン

1. **小さく確実に**
   - 1回の作業で1つの具体的改善
   - 完璧を求めすぎない
   - 動作する状態を保つ

2. **数学的洞察の記録**
   - 証明中に気づいた数学的構造
   - 予想外の依存関係
   - より一般的な定理の可能性

3. **Lean特有の発見**
   - 型システムが強制する厳密性
   - 計算可能性から得られる洞察
   - エラーを防ぐ型設計

### 成果の記録と Git コミット

#### 1. 作業内容の記録
作業完了後、以下を簡潔に記録：
```markdown
## 実施内容
- [具体的に何を達成したか]

## 数学的洞察
- [証明過程で発見した興味深い事実]

## 技術的メモ
- [Lean/mathlib 特有の注意点]

## 次の優先タスク
- [最も取り組むべき次の課題]
```

#### 2. 必須 Git コミット
**必ず最後に git コミットを実行**してください：

```bash
git add -A
git commit -m "Lean実装第N回: [具体的な達成内容]

- 解決したsorry: [ファイル名:行数]
- 修正したエラー: [エラーの種類]
- 新たな洞察: [発見した数学的構造]
- ビルド状態: [成功/失敗の詳細]

次回優先: [最重要な次のタスク]"
```

**コミットメッセージの例：**
```
Lean実装第3回: FactorialSeries.invFactorialTendsto を証明

- 解決したsorry: FactorialSeries.lean:42
- 修正したエラー: 型推論の曖昧性を明示的型注釈で解決
- 新たな洞察: 1/n! → 0 の証明でsqueeze定理が有効
- ビルド状態: FactorialSeriesモジュール完全成功

次回優先: HittingTime.lean:67のsorry (PMF正規化)
```

#### 3. 重要な注意
- あなたの変更は git diff で厳密に評価されます
- 「改善した」という主張と実際の変更が一致する必要があります
- 意味のない変更や後退は即座に発見されます
- コミットメッセージは次の実装者への重要な情報です
