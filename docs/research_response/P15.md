# Series Reindexing Using Indicator Functions in Lean 4

**Main Takeaway.**
If `f : ℕ → ℝ` is summable, then padding its values with zeros for the first two indices and shifting by two has no effect on the total sum:

```lean
theorem reindex_with_indicator {f : ℕ → ℝ} (h_summable : Summable f) :
  ∑' n, (if n ≥ 2 then f (n - 2) else 0) = ∑' k, f k
```


## 1. Indicator‐Function Decomposition

Define the “zero‐padded” version of `f`:

```lean
def f_indicator (f : ℕ → ℝ) : ℕ → ℝ :=
  fun n => if n ≥ 2 then f (n - 2) else 0
```

Basic lemmas:

```lean
lemma indicator_zero_below {f : ℕ → ℝ} {n : ℕ} (h : n < 2) :
  f_indicator f n = 0 := by
  simp [f_indicator, if_neg (not_le.mpr h)]
```


## 2. Restricting to the Subtype `{n // n ≥ 2}`

Since `f_indicator f n = 0` for `n = 0,1`, its support lies in `{n // 2 ≤ n}`.  By

```lean
tsum_subtype_eq_of_support_subset
  (fun n => f_indicator f n)
  (by
    intro n hn
    simp [Function.support, f_indicator] at hn
    split_ifs at hn with h₂
    · exact h₂
    · contradiction)
```

we may rewrite the infinite sum over `ℕ` as a sum over the subtype:

```lean
have : ∑' n : ℕ, f_indicator f n
       = ∑' k : {n // 2 ≤ n}, f_indicator f k := by
  apply tsum_subtype_eq_of_support_subset <;> simp [f_indicator]
```


## 3. Reindexing by the Bijection `k ↦ k+2`

Define the equivalence

```lean
def shift_bij : ℕ ≃ {n // 2 ≤ n} where
  toFun    k := ⟨k + 2, Nat.le_add_right _ _⟩
  invFun   n := n.val - 2
  left_inv k := by simp [Nat.sub_add_cancel (Nat.le_add_right 2 k)]
  right_inv n := by
    ext; simp [Nat.add_sub_cancel' n.prop]
```

Observe `shift_bij.invFun ⟨n, h⟩ = n.val - 2`.  Hence

```lean
have : ∑' k : {n // 2 ≤ n}, f_indicator f k
       = ∑' k : {n // 2 ≤ n}, f (k.val - 2) := by
  congr 1; funext k; rfl

... = ∑' m : ℕ, f m
```

where the last equality is `Equiv.tsum_eq shift_bij.symm h_summable`, noting that

```
f_indicator f (shift_bij.toFun m) = f (m + 2 - 2) = f m.
```


## 4. Complete Lean 4 Proof

```lean
import Mathlib.Topology.Algebra.InfiniteSum.Basic  -- hasSum, Summable, tsum_subtype_eq_of_support_subset
open Equiv Filter

def f_indicator (f : ℕ → ℝ) : ℕ → ℝ :=
  fun n => if n ≥ 2 then f (n - 2) else 0

theorem reindex_with_indicator {f : ℕ → ℝ} (h_summable : Summable f) :
    ∑' n, f_indicator f n = ∑' k, f k := by
  -- (1) Restrict to the subtype `n ≥ 2`
  have sum_subtype :
    ∑' n : ℕ, f_indicator f n
    = ∑' k : {n // 2 ≤ n}, f_indicator f k := by
    apply tsum_subtype_eq_of_support_subset
    · intro n hn
      simp [Function.support, f_indicator] at hn
      split_ifs at hn with h₂
      · exact h₂
      · contradiction
  rw [sum_subtype]
  -- (2) Reindex along the bijection `k ↦ k+2`
  refine Equiv.tsum_eq _ (h_summable)  -- will prove f_indicator f ∘ shift_bij = f
  dsimp
  funext m
  -- `shift_bij (m)` has `val = m+2`, so `f_indicator f (m+2) = f (m+2 - 2) = f m`
  simp [f_indicator, Nat.add_tsub_cancel_left]
```


## 5. Examples

```lean
-- Exponential series (factorial)
example (hf : Summable fun k => 1 / k.factorial) :
  ∑' n, if n ≥ 2 then 1/((n - 2).factorial) else 0 = exp 1 := by
  rw [reindex_with_indicator hf]
  exact exp_series_sum

-- Geometric series
example {r : ℝ} (hr : |r| < 1) :
  ∑' n, if n ≥ 2 then r^(n - 2) else 0 = 1 / (1 - r) := by
  rw [reindex_with_indicator (summable_geometric_of_lt_1 hr.1 hr.2)]
  exact tsum_geometric_of_lt_1 hr.1 hr.2
```

This completes a clean, indicator‐function–based formalization in Lean 4, leveraging `tsum_subtype_eq_of_support_subset` and `Equiv.tsum_eq` to handle zero‐padding and index‐shifting.
