# Missing Lean 4 v4.12.0 API Lemmas

Below are the v4.12.0 replacements, imports, and proof patterns for each of the four issues blocking your `FactorialSeries.lean` development.

## 1. `div_lt_one_iff_of_pos` Missing

**Replacement lemma:**

```lean
Nat.div_lt_one_iff
  : ∀ {a b : ℕ}, 0 < b → a / b < 1 ↔ a < b
```

**Import path:**

```lean
import Mathlib.Data.Nat.Basic
```

**Corrected proof syntax:**

```lean
import Mathlib.Data.Nat.Basic

-- Given h_pos : 0 < b
rw [Nat.div_lt_one_iff h_pos] at hn
```

**Alternative approach:**
If you have `a` and `b` in a more general `LinearOrderedField` (e.g. `ℝ`), use:

```lean
import Mathlib.Algebra.Order.Field.Basic

@[simp] theorem div_lt_one {a b : α} [LinearOrderedField α] (hb : 0 < b) :
    a / b < 1 ↔ a < b := by
  -- in Field.Basic you have the `div_lt_one` lemma
  simpa [one_mul] using (div_lt_iff_lt_mul _ hb).trans (by simp [one_mul])
```


## 2. `tendsto_order.mp` Issues

To derive `∀ᶠ n in atTop, f n < 1` from `h_tendsto : Tendsto f atTop (𝓝 0)`:

**Lemma to use:**

```lean
Filter.Tendsto.eventually
  : Tendsto f l m → ∀ s, s ∈ m → ∀ᶠ x in l, f x ∈ s
```

**Import path:**

```lean
import Mathlib.Topology.Algebra.Order.Limits
```

**Corrected proof syntax:**

```lean
import Mathlib.Topology.Algebra.Order.Limits
open Filter

-- Given h_tendsto : Tendsto f atTop (𝓝 0) and
--       one_pos    : 0 < (1 : α)

have h_eventually : ∀ᶠ n in atTop, f n < 1 := by
  -- `Iio_mem_nhds` says `Iio 1 ∈ 𝓝 0` whenever `0 < 1`
  have : (Iio (1 : α)) ∈ 𝓝 (0 : α) := Iio_mem_nhds one_pos
  -- apply `Tendsto.eventually`
  exact (h_tendsto.eventually this).mono fun _ => id
```


## 3. `tendsto_const_div_atTop_nhds_zero_nat` Missing

**Replacement lemma:**

```lean
tendsto_inverse_atTop_nhds_zero_nat
  : Tendsto (fun n : ℕ => (n : α)⁻¹) atTop (𝓝 0)
```

**Import path:**

```lean
import Mathlib.Analysis.SpecificLimits.Basic
```

**Corrected proof syntax:**

```lean
import Mathlib.Analysis.SpecificLimits.Basic
open Filter

theorem one_div_succ_tendsto_zero {α : Type*} [LinearOrderedField α] :
    Tendsto (fun n : ℕ => (1 : α) / (n + 1)) atTop (𝓝 0) := by
  -- rewrite `1 / (n + 1) = ((n + 1) : α)⁻¹`
  have : (fun n => (1 : α) / (n + 1)) = fun n => ((n + 1 : ℕ) : α)⁻¹ := by
    ext n; simp
  -- use `tendsto_inverse_atTop_nhds_zero_nat`
  rwa [this]
```


## 4. Factorial Field Operations

When you need to manipulate factorials inside functions or in a `LinearOrderedField`, always keep the factorial in `ℕ` and cast *after* computing:

```lean
import Mathlib.Data.Nat.Factorial
import Mathlib.Data.Real.Basic    -- or your chosen `α` field

open Nat

-- If you have (n + 1).factorial in a real-valued function:
def term (n : ℕ) : ℝ :=
  1 / ((n + 1 : ℕ).factorial : ℝ)

-- Or equivalently
def term' (n : ℕ) : ℝ :=
  (n + 1).factorial⁻¹    -- using `Fact` notation if you `open BigOperators` and 
                        -- provided you have `inv` on ℝ
```

**Key points:**

- Use `Nat.factorial` on `ℕ`.
- Cast to your field with `((...) : α)` *after* computing the factorial.
- If you need the inverse in the field, use `inv` or `⁻¹`.
