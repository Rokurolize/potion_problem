# Lean 4における定理探索とAPI検索の戦略

mathlib4における定理の発見と効率的なコードナビゲーションは、Lean 4を使用する際の中核的な課題の一つです[1]。本稿では、Lean 4のコード組織、効果的な検索戦略、および定理の可視性に関する包括的な分析を提供します。

## Lean 4のコード組織構造

### 定理の定義と使用の分離

Lean 4では、定理の定義場所と使用場所が必ずしも同一ではありません[2]。これは、プロジェクトの可読性と保守性を向上させるための意図的な設計選択です。具体的には以下のような構造になっています：

- **基礎定義**: コアライブラリやInit名前空間に配置
- **特化した定理**: 特定の数学分野に対応する名前空間に配置
- **補助定理**: より高レベルな証明で使用される、基礎となる補題


### `nonrec`キーワードと定理の可視性

`nonrec`キーワードは、定理の可視性に重要な影響を与えます[3]。このキーワードを使用することで、既存の再帰的定義を非再帰的に再定義することができますが、名前解決に影響を与える可能性があります。

### モジュール階層システム

mathlib4は、階層的なモジュール構造を採用しています[4][5]：

```
Mathlib/
├── Algebra/
│   ├── Group/
│   ├── Ring/
│   └── Field/
├── Analysis/
│   ├── Normed/
│   └── Metric/
├── Topology/
│   ├── Algebra/
│   │   └── InfiniteSum/
│   └── Basic/
└── Data/
    ├── Nat/
    └── List/
```


## 効果的な定理検索戦略

### 1. セマンティック検索エンジンの活用

#### LeanSearch

LeanSearchは、mathlib4用の包括的なセマンティック検索エンジンです[6][7][8]。このツールは以下の特徴を持ちます：

- **自然言語クエリ**: 非形式的なクエリを受け付け、関連する定理を見つけます
- **形式化支援**: 形式的な定理文を非形式的な記述に変換
- **コンテキスト理解**: 文脈を考慮した検索結果の提供


#### LeanExplore

LeanExploreは、Lean 4宣言用の検索エンジンです[9]。以下の機能を提供します：

- **ハイブリッドランキング**: セマンティック埋め込み、BM25+、PageRankを統合
- **多言語対応**: 形式的および非形式的な検索をサポート
- **Python API**: プログラマティックなアクセスが可能


### 2. 従来の検索手法

#### Loogle

Loogleは、Leanとmathlibの定義および定理を検索するツールです[10]：

```lean
-- 定数による検索
Real.sin

-- 部分式による検索
_ * (_ ^ _)

-- 結論による検索
|- tsum _ = _ * tsum _
```


#### `exact?`タクティク

`exact?`タクティクは、現在のゴールに適用可能な定理を自動的に見つけます[11][12]：

```lean
example (h : P) : P := by
  exact? -- hを提案
```


### 3. VS Code拡張機能の活用

Lean 4のVS Code拡張機能は、強力な定理探索機能を提供します[13][14]：

- **F12**: 定義への移動
- **Ctrl+Shift+F**: プロジェクト全体での検索
- **Ctrl+K Ctrl+S**: キーボードショートカットの表示[15]


### 4. コマンドライン検索ツール

#### `#help`コマンド

`#help`コマンドは、Leanの様々な側面をリスト化します[16][17]：

```lean
#help option    -- オプションをリスト化
#help attr      -- 属性をリスト化  
#help tactic    -- タクティクをリスト化
```


## `@summable_nat_add_iff`の適用構文分析

`@summable_nat_add_iff ℝ _ _ _ (fun i => (f i : ℝ)) k`という構文は、Lean 4における明示的な定理適用の例です[18][19]。この構文の構成要素：

- **`@`記号**: 暗黙引数を明示的に指定するためのプレフィックス[20]
- **型引数**: `ℝ`は実数型を指定
- **アンダースコア**: 推論可能な引数のプレースホルダー
- **関数引数**: `(fun i => (f i : ℝ))`は関数の明示的な型注釈


## mathlib4バージョン移行の課題

### バージョン間の定理変更

mathlib4では、バージョン間で定理の場所や名前が変更されることがあります[21]：

- **定理の移動**: ファイル間での定理の移動
- **名前の変更**: より一貫性のある命名規則への移行
- **名前空間の再構成**: より論理的な構造への変更


### 移行戦略

効果的な移行戦略には以下が含まれます[21]：

1. **段階的更新**: 一度にすべてを更新するのではなく、段階的にアップデート
2. **非推奨属性の活用**: `@[deprecated]`属性を持つ定理の識別
3. **`#min_imports`の使用**: 最小限のインポートの自動提案

## 命名規則と組織パターン

### mathlib4の命名規則

mathlib4では、以下の命名パターンを採用しています[2]：

- **snake_case**: 証明とPropの名前
- **UpperCamelCase**: 型と構造体
- **lowerCamelCase**: その他の値


### 定理発見のヒューリスティック

効果的な定理発見のためのヒューリスティック：

1. **型に基づく検索**: 関連する型名を含む検索
2. **操作に基づく検索**: `add_`, `mul_`, `sum_`などの操作プレフィックス
3. **プロパティに基づく検索**: `comm`, `assoc`, `inv`などのプロパティサフィックス

## 定理の可視性とアクセシビリティ

### インポート階層の理解

Lean 4では、インポート階層が定理の可視性を決定します：

```lean
import Mathlib.Topology.Algebra.InfiniteSum.Basic
-- このインポートにより、無限和に関する基本定理にアクセス可能
```


### 名前空間の管理

適切な名前空間の使用により、定理へのアクセスが容易になります：

```lean
open InfiniteSum  -- InfiniteSum名前空間を開く
open BigOperators -- 大きな演算子の記法を有効化
```


## 自動化ツールとタクティク

- **`simp`**: 簡約タクティク
