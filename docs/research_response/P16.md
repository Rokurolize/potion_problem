# Import Resolution Table

| Old Import | New Import | Notes |
| :-- | :-- | :-- |
| `import Mathlib.Order.Filter.AtTop` | **No longer exists** | Filter theory has been reorganized under `Order.Filter`—there is no direct `AtTop` module. |
| `import Mathlib.Order.Filter.AtTopBot` | `import Mathlib.Order.Filter.AtTopBot.Defs` | Defines `Filter.atTop` and `Filter.atBot` along with their basic lemmas and `Tendsto` instances[^1]. |
| → _General filter theory_ | `import Mathlib.Order.Filter.Basic` | Core definitions of `Filter`, `Eventually`, `Tendsto`, `map`, `comap`, etc.[^2]. |
| → _Override and helper defs_ | `import Mathlib.Order.Filter.Defs` | Low-level definitions; filters as lattice elements, `generate`, `principal`, `join`, `bind`, `pi`, `smallSets`[^3]. |
| `import Mathlib.Analysis.Asymptotics.Asymptotics` | `import Mathlib.Analysis.Asymptotics.Defs` | Most API moved to `Defs` submodule; contains basic asymptotic relations like `IsO`, `IsTheta`, `IsNegligible`[^4]. |
| `import Mathlib.Algebra.BigOperators.Basic` | `import Mathlib.Algebra.BigOperators.Group.Finset.Basic` | Big-operator notation and lemmas now live under `Group.Finset`[^5]. |

*Explanation of Reorganization*
Mathlib4 centralizes each theory in `Mathlib.<Area>.<Subarea>.Defs` or `Basic` files, splitting low-level definitions (`Defs`) from higher-level API. Filter theory now lives entirely under `Mathlib.Order.Filter`, with:

- **Defs**: foundational definitions (`Filter`, `principal`, `generate`, etc.).
- **Basic**: user-facing API (`Eventually`, `Tendsto`, `map`, `comap`, product/coproduct filters).
- **AtTopBot.Defs**: definitions and lemmas for limits to ±∞ (`Filter.atTop`, `Filter.atBot`).


# API Migration Guide

## 1. `tsum_equiv` → `Equiv.tsum_eq`

```lean
-- OLD
#check tsum_equiv

-- NEW
import Mathlib.Topology.Algebra.InfiniteSum.Defs
#check Equiv.tsum_eq
```

The lemma `Equiv.tsum_eq` states:

```lean
theorem Equiv.tsum_eq {α β M} [AddCommMonoid M]
  {e : α ≃ β} {f : β → M} (hf : Summable f) :
  ∑' b, f b = ∑' a, f (e a)
```

and replaces `tsum_equiv` directly[^6][^7].

## 2. `hasSum_iff_tendsto_nat_of_summable` → `hasSum_iff_tendsto_sum_nat`

```lean
-- OLD
#check hasSum_iff_tendsto_nat_of_summable

-- NEW
import Mathlib.Topology.Algebra.InfiniteSum.Defs
#check hasSum_iff_tendsto_sum_nat
```

Use:

```lean
have h' := hf.hasSum
rw [hasSum_iff_tendsto_sum_nat] at h'
```

where `hasSum_iff_tendsto_sum_nat` equates `HasSum f a` with `Tendsto (s ↦ ∑ x in s, f x) atTop (𝓝 a)`[^7].

## 4. Neighborhood filter `𝓝` → `nhds`

```lean
-- OLD
#check 𝓝 (0 : ℝ)

-- NEW
#check nhds (0 : ℝ)
```

Replace all occurrences of `𝓝 x` with `nhds x`[^7].

## 5. Sum notation changes

```lean
-- OLD
∑ x in s, f x

-- NEW
∑ x ∈ s, f x
```

The linter warns on `in`; update to `∈` for all `Finset` sums and products[^8].

## 6. `zero_pow` type mismatch

```lean
-- Solution using `Nat.pos_iff_ne_zero`
lemma zero_pow_of_pos {n : ℕ} (hn : n > 0) : (0 : ℝ) ^ n = 0 := by
  exact zero_pow (Nat.pos_iff_ne_zero.mp hn)

-- OR pattern match
lemma zero_pow_of_pos' {n : ℕ} (hn : n > 0) : (0 : ℝ) ^ n = 0 := by
  cases n with
  | zero   => contradiction
  | succ n => exact zero_pow_succ n
```


# Working Implementations

```lean
import Mathlib.Topology.Algebra.InfiniteSum.Defs
import Mathlib.Data.Equiv.Basic
import Mathlib.Order.Filter.Basic

namespace Custom

-- tsum_equiv replacement
theorem tsum_equiv {α β M} [AddCommMonoid M]
  {f : β → M} (e : α ≃ β) (hf : Summable f) :
  ∑' a, f (e a) = ∑' b, f b := by
  rw [Equiv.tsum_eq hf]

-- hasSum_iff_tendsto_nat_of_summable replacement
theorem hasSum_tendsto_nat {α M} [AddCommMonoid M] [TopologicalSpace M]
  {f : α → M} {a : M} (h : Summable f) :
  HasSum f a ↔ Tendsto (fun s : Finset α => ∑ x in s, f x) atTop (𝓝 a) := by
  exact hasSum_iff_tendsto_sum_nat

end Custom
```
