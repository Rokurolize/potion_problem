# 最適化実装戦略提案：v4.12.0へのバックポートによる望遠級数証明

**最終結論：部分集合分離戦略（戦略1）を第一優先とし、階乗級数直接計算（戦略2）を補完的に実装する。Phase Aで部分集合分離のプロトタイプを1–2日で構築し、Phase Bで階乗級数アプローチを3–5日で完全実装、最後にPhase Cで統合テスト1日。**

## 1. 戦略評価と優先順位付け

| 評価観点 | 戦略1：部分集合分離 | 戦略2：階乗級数直接計算 |
| :-- | :-- | :-- |
| 数学的妥当性 | ∀Summable f, ∑(f n - f (n+1))=f 0 を一般的に証明可能 | n≥2 のみを扱う特例証明 |
| 実装複雑度 | v4.22.0-rc3 API依存少なく、型調整も微小 | 条件分岐（if n≥2）と factorial 操作の明示実装 |
| 成功確率 | 高：Multipliable.tprod_subtype 系を模倣可能 | 中：直接計算ゆえ収束保証を追加実装要 |
| 確率論文脈での優位性 | 一般定理として他級数にも流用可 | 媚薬問題固有、汎用性低 |

→ **戦略1を第一優先**。戦略2は「階乗級数＝確率質量関数」の直接証明段階で補完的に採用。

## 2. リスク評価と軽減策

| リスク項目 | 軽減策 |
| :-- | :-- |

| 収束保証 API (`tprod_vanishing`) の型差異 | `Filter.tendsto` と `Summable` の既存定理を組み合わせて代替構築 |

| 証明失敗時の全体停止 | 部分ごとに `example` タグ付きテストを導入し、段階的に CI 上で検証 |

## 4. 数学的厳密性の保証

- **型安全性**：v4.22.0-rc3 の型クラス制約は踏襲せず、v4.12.0 の `Multipliable`・`Summable`・`Filter.tendsto` の既存 API で完結。
- **証明厳密性**：近似やトリックを用いず、すべてのステップで `by` 構文による完全機械検証を実施。
- **再利用性**：一般化した補助定理を `Mathlib4` の命名規則に従い命名し、他級数証明でも活用可能な設計。
