# Hopf shuffle対数解

この解では、生存prefix全体をshuffle Hopf代数の群的元として見て、その対数が1文字だけに潰れることから指数関数を読む。

独立一様乱数`U_1,U_2,...`について

```text
S_n = U_1 + ... + U_n,
tau = min { n >= 1 : S_n > 1 }
```

とする。期待値は

```text
E[tau] = sum_{n >= 0} P(S_n <= 1)
```

である。

## 生存prefixを語として見る

`S_n <= 1`なら、累積時刻

```text
0 <= t_1 <= ... <= t_n <= 1
```

を持つ。これは区間`[0,1]`上の1文字形式`dt`の長さ`n`の反復積分である。

1文字`a=dt`だけを持つshuffle代数で、すべての長さの生存prefixを形式的に

```text
G = 1 + a + a^{shuffle 2}/2! + a^{shuffle 3}/3! + ...
```

として束ねる。反復積分で評価すると、長さ`n`の項はちょうど

```text
int_{0 <= t_1 <= ... <= t_n <= 1} dt_1 ... dt_n
```

であり、`P(S_n <= 1)`に等しい。

## shuffle対数

shuffle Hopf代数で、反復積分の全和は群的元である。群的元の対数は原始元になる。ここでは文字が1つだけなので、原始元として残るのは

```text
a
```

だけである。つまり

```text
log_shuffle G = a,
G = exp_shuffle(a).
```

区間`[0,1]`で評価すると、shuffle指数の評価は通常の指数になる。

```text
E[tau]
  = int G
  = exp(int_0^1 dt)
  = e.
```

この証明では、各単体体積を個別に計算しない。生存prefixの全次数をHopf代数の群的元としてまとめると、shuffle対数が1文字の原始元に落ちる。`e`は、その原始元の積分値`1`を指数へ戻した値として現れる。
