  ### **処方箋1：無限和の分解には `tsum_subtype` を使う**

  AIは `(∑' k, if k > n then ...)` 
  という条件付き無限和の扱いに最も苦労しています。これを解決する最短の道は、全単射（bijection）
  を手動で構築することでは**ありません**。より抽象的な「部分型（subtype）」の概念を利用します。

  **最重要API**: `tsum_subtype_add_tsum_subtype_compl`

  これは `api-library.md` にも記載されている、まさにこの問題のために存在する定理です。

  ```lean
  -- Summable f → (∑' (x : {x // p x}), f x) + (∑' (x : {x // ¬p x}), f x) = ∑' x, f x
  -- 意味：ある条件pを満たす元の和と、満たさない元の和を足すと、全体の和に等しい。
  ```

  これを利用した証明の骨格は以下のようになります。

  ```lean
  theorem tail_probability_formula (n : ℕ) :
      (∑' k, if k > n then hitting_time_pmf k else 0) = 1 / n.factorial := by

    -- ステップ1: if文を、部分型{k | k > n}上の和に変換する
    -- これには tsum_eq_tsum_of_ne_zero を使うのが定石
    have h_tail_eq_subtype :
        (∑' k, if k > n then hitting_time_pmf k else 0) = ∑' (k : {k // k > n}), 
  hitting_time_pmf k := by
      -- tsum_eq_tsum_of_ne_zero を使い、k > n でない項が0になることを示す
      apply tsum_eq_tsum_of_ne_zero
      intro k
      split_ifs with h_gt
      · -- k > n の場合、値が0でないことを示す（これは実際には不要）
        simp
      · -- k ≤ n の場合、値が0であることを示す
        simp

    rw [h_tail_eq_subtype]

    -- ステップ2: 補集合の和を使って、全体の和から有限和を引く形に変形する
    -- (∑' k ∈ {k>n}) = (∑' k) - (∑' k ∈ {k≤n})
    have h_decomp := (tsum_subtype_add_tsum_subtype_compl pmf_summable ({k | k > n})).symm
    rw [h_decomp, pmf_sum_eq_one]
  ```

  ### **処方箋2：有限和の計算は `Finset.sum_sdiff` で分割する**

  AIは `∑_{k=2 to n}` の部分を計算するために `sum_bij` 
  を使って泥沼にはまっています。エラーログの多く（`introN` の失敗、`omega` 
  の失敗）はこれが原因です。これも、もっと簡単なAPIがあります。

  **推奨API**: `Finset.sum_sdiff`

  和の範囲 `Finset.range (n + 1)` を、値が `0` になる部分 (`Finset.range 2`) と、そうでない部分 
  (`Finset.range (n + 1) \ Finset.range 2`) に分割します。

  ```lean
  -- n ≥ 2 のケース
  have h_sum_finite : ∑ k in Finset.range (n + 1), hitting_time_pmf k = 1 - 1 / n.factorial := 
  by
    -- ゴール: ∑_{k=0 to n} pmf k = 1 - 1/n!

    -- ステップ1: 和を k=0,1 の部分と k≥2 の部分に分割する
    have h_sub : Finset.range 2 ⊆ Finset.range (n + 1) := Finset.range_mono (by omega)
    rw [← Finset.sum_sdiff h_sub]

    -- ステップ2: k=0,1 の和は0なので消す
    -- `prob_tau_eq_zero_one` を使って、この部分の和が0になることを示す
    have h_zero_sum : ∑ k in Finset.range 2, hitting_time_pmf k = 0 := by
      simp [Finset.sum_range_succ, prob_tau_eq_zero_one]
    rw [h_zero_sum, add_zero]
    -- これでゴールの左辺は `∑ k in Finset.range (n + 1) \ Finset.range 2, ...` となる

    -- ステップ3: 残った和 `∑_{k=2 to n}` をテレスコープ和の公式で計算する
    -- この部分は `pmf_telescoping` を使い、帰納法や専用API `Finset.sum_range_sub_adjacent`

    -- この部分はAIの試行錯誤の中でも成功に近い部分なので、再利用できる
    -- ...
  ```
