# 重み付きprefix分配関数解

この解では、停止前prefixをただ数えるのではなく、まず各prefixに重みを付けた分配関数としてまとめる。

独立な一様乱数`U_1,U_2,...`について

```text
S_0 = 0,
S_n = U_1 + ... + U_n,
tau = min { n >= 1 : S_n > 1 }
```

と置く。停止前prefixの個数は

```text
tau = sum_{n >= 0} 1_{S_n <= 1}
```

なので、求める値は

```text
E[tau] = sum_{n >= 0} P(S_n <= 1)
```

である。

## prefixに重みを付ける

残り容量が`r`の状態から、停止前prefix全体を数える分配関数を

```text
Z_z(r)
```

と書く。ただし、深さ`n`のprefixには重み`z^n`を掛ける。空prefixは深さ`0`なので、必ず重み`1`で寄与する。

残り容量`r`から一口`u`を飲んでなお生きているには

```text
0 <= u <= r
```

でなければならない。この一歩で深さが`1`増えるので、以後の部分木には重み`z`が掛かる。したがって

```text
Z_z(r) = 1 + z int_0^r Z_z(r-u) du.
```

変数を替えれば

```text
Z_z(r) = 1 + z int_0^r Z_z(v) dv.        (1)
```

これは、各`P(S_n<=1)`を先に出すのではなく、全ての停止前prefixを深さで重み付けした総量を一度に閉じる式である。

## 重み付き総量を解く

(1)を微分すると

```text
Z_z'(r) = z Z_z(r),
Z_z(0) = 1.
```

よって

```text
Z_z(r) = exp(zr).
```

初期容量は`1`なので

```text
Z_z(1) = exp(z).
```

一方、定義から

```text
Z_z(1) = sum_{n >= 0} z^n P(S_n <= 1).
```

したがって、停止前prefixを普通に数えるには`z=1`を代入すればよい。

```text
E[tau]
= sum_{n >= 0} P(S_n <= 1)
= Z_1(1)
= e.
```

この証明の要点は、期待値をいきなり解くのではなく、深さを覚えている分配関数を先に作るところにある。重み`z`は一口進むたびに一度だけ掛かり、その自己再生式が`Z_z'=zZ_z`を与える。最後に`z=1`へ戻すと、停止前prefixの総量がそのまま`e`になる。
