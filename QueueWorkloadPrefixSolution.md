# 待ち行列仕事量prefix解

この解では、飲用列を一人のサーバへ到着する仕事量列として読む。

各一口の量を

```text
X_1, X_2, ...
```

とし、累積仕事量を

```text
W_n = X_1 + ... + X_n
```

とする。サーバが処理できる総仕事量を`1`と決めると、`W_n <= 1`であるprefixは、容量`1`の仕事窓に収まる投入履歴である。停止時刻`tau`は、初めて窓をあふれさせる投入番号である。

したがって尾和公式から

```text
E[tau] = sum_{n >= 0} P(W_n <= 1).
```

右辺は、容量`1`の窓に収まる全prefix履歴の期待個数である。

## 容量を動かした仕事窓

容量を`r`にしたときの全prefix履歴の質量を

```text
Q(r) = sum_{n >= 0} P(W_n <= r)
```

とする。空投入履歴があるので

```text
Q(0) = 1
```

である。

最後の投入量を`u`とする。`0 <= u <= r`なら、その手前の履歴は容量`r-u`の窓に収まっていればよい。一様密度は`1`なので、非空prefixの総質量は

```text
int_0^r Q(r-u) du
```

である。よって

```text
Q(r) = 1 + int_0^r Q(r-u) du
     = 1 + int_0^r Q(v) dv.
```

微分して

```text
Q'(r) = Q(r),   Q(0)=1.
```

したがって

```text
Q(r) = exp(r)
```

であり、

```text
E[tau] = Q(1) = e.
```

この証明では、停止時刻の分布を直接求めない。容量`r`の仕事窓に入る全投入履歴を一つの仕事量prefix空間として束ね、最後の仕事量を剥がす再帰だけで指数関数を読む。
