# Audience ladder solutions

このファイルはpotion problemの読者別説明集です。Lean側の窓口は
[`PotionProblem/AudienceLadder.lean`](PotionProblem/AudienceLadder.lean)に置いています。

各節は、その節だけを読んでも読めることを条件にします。同じ問題を何度も定義しているのは、品質ゲートで「その節だけを読む」ためです。

改稿方針:

- 低い段では、証明の全体ではなく、終わる回数をどう数えるかを先に理解させる。
- 「平均」「生き残る」「tau」「質量」「分配関数」のような語は、出す節で必ず言い換える。
- 高い段では圧縮してよいが、圧縮対象の記号は節内で定義する。
- 完了判定はロールプレイ品質ゲートで行う。各読者のレビュアーが、その節だけを読んで`わかった`と答える必要がある。

## 1. 未就学児でも理解できる解法

想定読者: 6歳前後。保育所・幼稚園修了直後。分数、小数、筆算、記号式はまだ使わず、コップ、線、回数、多い・少ないで考える。

小さなコップがある。コップには「ここまでならまだ大丈夫」という線がある。その線を`1`と呼ぶ。

毎回、袋からカードを1枚引く。カードには「すこし」「まんなかくらい」「たくさん」のような量が書いてある。小さい量も大きい量も、同じくらい出る。その分だけコップに入れる。線をこえたら終わり。入れた回数を数える。

たとえばこうなる。

```text
ほとんどいっぱいまで入れる。まだ線をこえない。
次にまたたくさん入れる。線をこえたので終わり。
このときは2回。
```

別の日はこうなる。

```text
ちょっとだけ入れる。まだ大丈夫。
またちょっとだけ入れる。まだ大丈夫。
またちょっとだけ入れる。まだ大丈夫。
最後にたくさん入れる。ここで線をこえる。
このときは4回。
```

だから、答えはいつも2回ではない。3回の日も4回の日もある。

ここでは「みんなで何回くらいだったか」と呼ぶ。

たとえば、5人で遊ぶ。2回で終わった子もいる。3回で終わった子もいる。4回で終わった子もいる。みんなを見て「だいたい何回かな」と考えると、2回より多くて、3回より少し少ないくらいになる。

大人が細かく数えるときは、こういうシールの数え方をする。

```text
はじめに残っているしるし: みんな1枚
1回入れたあとも残っているしるし: みんな1枚
2回入れたあとも残っているしるし: だいぶ少ない
3回入れたあとも残っているしるし: もっと少ない
```

「4回で終わった人」は、はじめ、1回あと、2回あと、3回あとにしるしを置くので、しるしが4枚になる。つまり、しるしの枚数がその人の回数と同じになる。

だから、しるしをたくさん集めると、「みんなで何回くらいだったか」が見えてくる。

この遊びを本当にたくさんやると、「みんなで何回くらいだったか」は`2回`より大きく、`3回`より少し小さい。この数に、大人は`e`という短い名前を付けている。ここでは、`e`を計算しなくてよい。「2と3のあいだで、3より少し小さい数の名前」と思えばよい。

この段で持ち帰ること:

```text
こぼれるまでの回数は日によって変わる。
みんなで何回くらいかを見る。
この遊びは2回と3回の間で、3回より少し小さいくらい。
その数の名前がe。
```

```lean
preschoolChildSolution
```

## 2. 小卒でも理解できる解法

想定読者: 12歳前後。小学校6年修了直後の児童。小数、分数、割合、面積、表は使えるが、中学の負の数、文字式、方程式、関数、形式的証明はまだ使わない。

ゲームの約束はこうする。

```text
コップの限界を1とする。
毎回、0から1までの量をランダムに足す。
合計が1をこえたら終わり。
終わるまでに足した回数を答えにする。
```

「まだ残る」とは、まだ合計が`1`をこえていないという意味。

平均回数を出すには、1人ずつの回数を直接足してもよい。でも、もっと見やすい数え方がある。各人にシールを置いてもらう。

```text
スタート時点でシールを1枚置く。
1回足しても残っていたら、また1枚置く。
2回足しても残っていたら、また1枚置く。
同じように、終わる直前まで置く。
```

ある人が4回目で終わったなら、その人は

```text
スタート
1回後
2回後
3回後
```

に1枚ずつ置く。合計4枚。つまり、その人のシールの枚数は、その人が足した回数と同じ。

全員のシールを段ごとに数えると、平均回数が出る。

```text
スタート時点: 全員いるので1
1回後: 1回ではまだ1をこえないので1
2回後: だいたい半分が残るので1/2
3回後: 2回後より少ない人が残る
4回後: 3回後よりもっと少ない人が残る
```

`2回後`が半分になる理由を絵で見る。紙の横を1回目の量、縦を2回目の量にする。横も縦も`0`から`1`まで。たとえば1回目が`0.2`、2回目が`0.7`なら、紙の中の「横0.2、縦0.7」の場所に点を打つ。

点が打てる場所全体は正方形になる。その中で、横と縦を足して`1`以下の場所だけが「2回後も残る」場所。これは斜めの線で切った三角形で、正方形のちょうど半分になる。

`3回後`や`4回後`も、大人が同じ数え方で計算できる。小卒の段では、その細かい計算を自分で出せなくてもよい。ここで大事なのは、段ごとに「まだ残る人の割合」を表にして、それを足すと平均回数になる、という数え方である。

だから平均は

```text
1 + 1 + 1/2 + もっと小さい分 + さらに小さい分 + ...
```

になる。この足し算の答えが`2.718...`で、名前は`e`。

この段で持ち帰ること:

```text
何回で終わったかを数える代わりに、
各段でまだ残っている人の割合を足してよい。
その足し算が平均回数になる。
```

```lean
elementaryGraduateSolution
```

## 3. 中卒でも理解できる解法

想定読者: 15歳前後。中学校3年修了直後。正負の数、文字式、関数、簡単な確率は使えるが、微積分、無限級数、測度論は使わない。

毎回足す量を

```text
U_1, U_2, U_3, ...
```

と書く。どの`U_i`も`0`から`1`までのランダムな数。途中までの合計を

```text
S_1 = U_1
S_2 = U_1 + U_2
S_3 = U_1 + U_2 + U_3
```

のように書く。

`n`回足してもまだ終わらないとは、

```text
S_n <= 1
```

という意味。ちょうど`0`が出るような境目は、ランダムに選ぶとまず起きないものとして考えてよい。このとき途中合計は必ず

```text
0 < S_1 < S_2 < ... < S_n <= 1
```

と小さい順に並ぶ。

ここで、`n`個の点を`0`から`1`の間に置くことを考える。点に名前を付けると、並び方は全部で

```text
n! = n * (n-1) * ... * 2 * 1
```

通りある。たとえば`3!`は`3*2*1=6`という意味。

```text
2個なら A<B と B<A の2通り。
3個なら ABC, ACB, BAC, BCA, CAB, CBA の6通り。
```

小さい順にきれいに並ぶ並び方は、そのうち1通りだけ。だから「途中合計が小さい順に並んで`1`以内にいる場所」は、全体の`1/n!`になる。

したがって

```text
n回後もまだ残る確率 = 1/n!
```

平均回数は、まだ残る確率を段ごとに足せばよい。理由は、たとえば4回目で終わった人なら

```text
0回後は残っている
1回後も残っている
2回後も残っている
3回後も残っている
4回後はもう残っていない
```

となり、「残っている」と答える段が4個あるから。その4個が、その人の回数そのものになる。

だから平均回数は、

```text
0回後に残る確率
1回後に残る確率
2回後に残る確率
3回後に残る確率
...
```

を順に足したものになる。各段の確率は`1/n!`で決まるので、あとは次の高卒段で、この足し算の名前を指数関数として読む。

Lean hook:

```lean
juniorHighGraduateSolution
```

## 4. 高卒でも理解できる解法

想定読者: 18歳前後。高校数学を数学IIIまで履修した卒業直後。指数関数、微分積分、数列・級数の入口は使えるが、測度論、抽象確率、Jacobian、行列式前提の体積変換は使わない。

毎回の増分を独立な一様乱数`U_1,U_2,...`とする。各`U_i`は`[0,1]`から一様に選ぶ。部分和を

```text
S_n = U_1 + ... + U_n
```

と置く。停止時刻を

```text
tau = min { n >= 1 : S_n > 1 }
```

と定義する。つまり`tau`は、合計が初めて`1`をこえる回数。

`tau > n`とは、`n`回足した後もまだ終わっていないという意味なので、

```text
tau > n  <=>  S_n <= 1.
```

`S_n <= 1`になる領域を、累積時刻

```text
t_1=S_1, t_2=S_2, ..., t_n=S_n
```

で見る。`U_i=0`のようにちょうど境目に乗る場合は確率0なので、確率や面積を計算するときは無視してよい。すると条件は

```text
0<t_1<...<t_n<=1
```

になる。この変換が面積や体積を変えないことは、低い次元で見るとわかる。たとえば`n=2`なら

```text
t_1=U_1,
t_2=U_1+U_2.
```

`U_1`を少し増やすと、`t_1`と`t_2`が同じだけ動く。これは斜め右上への動き。`U_2`を少し増やすと、`t_2`だけが動く。これは上への動き。この二つの動きでできる小さな平行四辺形は、もとの小さな長方形を横にずらした形で、底辺も高さも同じだから面積は変わらない。

逆向きに書けば

```text
U_1=t_1,
U_2=t_2-t_1.
```

なので、情報もつぶれていない。一般の`n`でも、前の座標を足して後ろの座標を作るだけなので、小さな箱を斜めにずらすだけで体積は変わらない。この領域の体積が`1/n!`になる理由は、次の通り。

`[0,1]^n`の体積は`1`。その中では、`n`個の座標の大小順は`n!`通りある。座標がたまたま等しくなる境界は面積も体積も`0`なので無視できる。独立一様なので、どの大小順の領域も同じ体積を持つ。したがって、特定の一つの順序

```text
0<t_1<...<t_n<=1
```

を満たす領域の体積は`1/n!`。よって

```text
P(tau > n) = 1/n!    (n >= 0).
```

非負整数値の回数では、期待値は尾確率の和で出せる。理由は、`tau=4`なら

```text
1_{tau>0}, 1_{tau>1}, 1_{tau>2}, 1_{tau>3}
```

の4個が`1`になり、それ以降は`0`になるから。

したがって

```text
E[tau] = P(tau>0) + P(tau>1) + P(tau>2) + ...
       = 1 + 1 + 1/2! + 1/3! + ...
```

指数関数の基本公式

```text
e^x = sum_{n>=0} x^n/n!
```

に`x=1`を入れると

```text
e = 1 + 1 + 1/2! + 1/3! + ...
```

なので

```text
E[tau] = e.
```

```lean

```

## 5. 大卒でも理解できる解法

想定読者: 22歳前後。数学、統計、物理、工学、情報系などの学部卒。確率変数、PMF、期待値、微積分、線形代数、変数変換は使える。

`U_i`を独立同分布の`Uniform(0,1)`とし、

```text
S_n = U_1 + ... + U_n,
tau = min { n >= 1 : S_n > 1 }
```

と置く。

まず尾確率を求める。`tau>n`は`S_n<=1`と同値。変数変換

```text
t_i = S_i
```

はJacobianが`1`で、領域を

```text
0 < t_1 < ... < t_n <= 1
```

へ送る。この単体の体積は`1/n!`なので

```text

= (n-1)/n!      (n >= 2).

       = sum_{n>=2} n(n-1)/n!
       = sum_{n>=2} 1/(n-2)!.
```

```text

```

```text
m'(r) = m(r).
```

```text
m(0)=1.
```
