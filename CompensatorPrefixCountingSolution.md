# compensator prefix counting解

この解では、停止前prefixの数を、容量軸上の決定論的補償項として読む。

残り容量を`x`とし、空prefixを含む期待prefix数を`V(x)`とする。容量`x`で次の入力を読むと、`u<=x`の部分だけが続行に寄与するので

```text
V(x)=1+int_0^x V(x-u)du.
```

この積分項は、容量軸上で「次のprefixが生まれる率」が1であることの補償項である。変数を替えると

```text
V(x)=1+int_0^x V(t)dt.
```

従って補償項の密度は現在のポテンシャルそのもので、

```text
V'(x)=V(x),  V(0)=1.
```

境界`x=0`では空prefixだけが残るため初期値は`1`である。よって

```text
V(x)=e^x.
```

元の初期容量は`1`なので

```text
E[tau]=V(1)=e.
```

この見方では、停止前prefixの発生数を容量に沿う補償過程として捉え、その補償密度が自己再生することから指数関数が出る。
