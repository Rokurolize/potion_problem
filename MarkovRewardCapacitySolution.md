# 残り容量Markov報酬解

この解では、停止時刻の分布を追わず、残り容量を状態とするMarkov過程に「生存prefix一つにつき報酬1」を払わせる。

独立な一様乱数`U_1,U_2,...`について

```text
S_0 = 0,
S_n = U_1 + ... + U_n,
tau = min { n >= 1 : S_n > 1 }
```

とする。停止前prefixを数えれば

```text
tau = sum_{n >= 0} 1_{S_n <= 1}
```

なので、求めたいのは停止までに得る報酬の期待総額である。

## 残り容量の報酬関数

残り容量が`x`ある状態から始めたとき、これから得る報酬の期待総額を

```text
R(x)
```

と書く。ここで、現在のprefixはまだ停止前なので、まず報酬`1`を受け取る。

次の一口が`u`なら、`u <= x`のときだけ生き残り、残り容量は`x-u`になる。`u > x`ならそこで停止し、以後の報酬はない。

`0 <= x <= 1`では一様密度が`1`なので、Markov報酬の一段分解は

```text
R(x) = 1 + int_0^x R(x-u) du.
```

変数を替えると

```text
R(x) = 1 + int_0^x R(v) dv.        (1)
```

この式では、停止時刻の確率質量関数を出していない。状態`x`にいる限り報酬`1`を払い、次状態へ進める部分だけを積分している。

## 報酬方程式を解く

(1)を微分して

```text
R'(x) = R(x),
R(0) = 1
```

を得る。したがって

```text
R(x) = exp(x).
```

初期の残り容量は`1`だから

```text
E[tau] = R(1) = e.
```

この証明の要点は、`tau`を「何回目で止まるか」ではなく「停止までに支払われる単位報酬の総額」として見るところにある。一様性は次状態への密度を`1`にし、独立性は残り容量`x-u`から同じ報酬関数`R`を再利用できることに現れる。
