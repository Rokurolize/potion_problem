# 最小DFA prefix解

この解では、停止前prefixを受理する最小DFAの重み付き受理量として読む。

独立一様乱数`U_1,U_2,...`について

```text
S_n = U_1 + ... + U_n,
tau = min { n >= 1 : S_n > 1 }
```

とする。状態を残り容量`r`とし、`r>=0`なら受理状態、次の一口で`r-U<0`になれば死状態へ落ちる、と考える。境界`x`までの受理prefix総量を

```text
F(x)=sum_{n>=0}P(S_n<=x)
```

と置く。`0<=x<=1`では、受理状態から次の受理状態へ進む遷移密度は常に`1`である。したがって受理prefixの重みは

```text
F'(x)=F(x),  F(0)=1
```

を満たす。

よって

```text
F(x)=e^x,
E[tau]=F(1)=e.
```

このDFAは有限状態ではなく連続状態だが、最小化すると必要な情報は残り容量だけである。答えは、その受理prefix言語の重み付き総量である。
