# Yoneda capacity functor解

この解では、容量`x`に対する期待prefix数を関手として見て、自然性が指数関数を決める。

容量`x`での停止前prefix期待数を

```text
F(x)=sum_{n>=0}P(U_1+...+U_n<=x)
```

とする。元の答えは`F(1)`である。容量を少し増やす写像`x -> x+dx`に対して、新しく現れるprefixの密度は、それ以前のprefix関手の値そのものになる。最後の入力で切れば厳密に

```text
F'(x)=F(x),  F(0)=1.
```

これは、容量区間の包含に沿ったprefix関手が、端点評価で表現されることを意味する。Yoneda的に言えば、容量`x`で観測される全prefixは、空prefixからの一意な拡張で決まり、その自然変換の値が指数関数である。

したがって

```text
F(x)=e^x
```

であり、

```text
E[tau]=F(1)=e.
```

名前だけの圏論ではなく、容量の増加に対する自然な制限・拡張が微分方程式を強制している。
