  ### ヒント1：無限和の分解には、もっと直接的なAPIを使う

  AIは無限和を「有限部分」と「尾部」に分けるのに苦労しています
  。`Summable.sum_add_tsum_nat_add` 
  を使おうとしていますが、もっとゴールに直結するAPIがあります。

  **最重要API:** `tsum_eq_sum_add_tsum_compl`

  この定理は、まさにやりたいことを実現してくれます。

  ```lean
  -- {α : Type*} [AddCommGroup α] [TopologicalSpace α] [T3Space
   α]
  -- {f : ℕ → α} (hf : Summable f) (s : Finset ℕ)
  tsum_eq_sum_add_tsum_compl (hf : Summable f) (s : Finset ℕ) :
    ∑' n, f n = (∑ n ∈ s, f n) + ∑' n, ইন্ডিকেটর {n | n ∉ s} f n
  ```
  （※ `indicator` の部分は実際には `subtype` 
  を使った和ですが、意味的にはこれと等価です）

  これを使えば、AIが`sorry`でごまかしていた証明の最初の部分が一
  撃で片付きます。

  **証明の方針:**

  1.  目標の式 `(∑' k, if k > n then hitting_time_pmf k else 0)
   = 1 / n.factorial` を変形し、`1 - (∑ k ∈ Finset.range (n + 
  1), hitting_time_pmf k) = 1 / n.factorial` にします。これは 
  `add_comm` や `eq_sub_iff_add_eq` などで簡単にできます。
  2.  そのためには、まず `(∑' k, if k > n then ...)` が `(∑' k,
   pmf k) - (∑ k in Finset.range (n+1), pmf k)` 
  と等しいことを示す必要があります。
  3.  ここで `tsum_eq_sum_add_tsum_compl` を `s := Finset.range
   (n + 1)` として適用します。すると、
      `∑' k, pmf k = (∑ k ∈ Finset.range (n + 1), pmf k) + (∑' 
  k ∈ {k | k > n}, pmf k)`
      という式が得られます。
  4.  あとは `(∑' k ∈ {k | k > n}, pmf k)` が `(∑' k, if k > n 
  then pmf k else 0)` と等しいことを示せば完了です。これには 
  `tsum_subtype_eq_if` が使えます。

  この方針なら、複雑な全単射の証明は一切不要です。

  ### ヒント2：有限和の添え字変換は、より構造的に行う

  AIは `∑_{k=2 to n}` を `∑_{j=0 to n-2}` に変換するために 
  `Finset.sum_bij` を使おうとして、引数の多さと複雑な証明に苦し
  んでいます。これももっと簡単な方法があります。

  **推奨されるアプローチ:** `Finset.sum_sdiff` と 
  `Finset.sum_image` を組み合わせる

  1.  まず、和の範囲から不要な部分を取り除きます。
      ```lean
      -- h_sub: Finset.range 2 ⊆ Finset.range (n + 1)
      rw [← Finset.sum_sdiff h_sub]
      -- これで和は ∑ k ∈ Finset.range (n + 1) \ Finset.range 
  2, ... となる
      ```
  2.  `hitting_time_pmf` の最初の2項は0なので、`sum` の中の `0`
   を `simp` で消します。
  3.  次に、添え字を変換します。`Finset.sum_image` 
  を使うと、全射・単射の証明が少し楽になります。
      ```lean
      -- 変換: e : ℕ → ℕ, e j := j + 2
      -- 変換後の和: ∑ j ∈ Finset.range (n-1), hitting_time_pmf
   (j + 2)
      -- を示すために、
      -- Finset.sum_image を使い、2つの集合が e 
  によって対応することを示す。
      ```

  **もっと簡単な別解:**

  `pmf_telescoping` を使って、和がテレスコープ和になることを利
  用します。これはAIも試みていますが、`sum_range_succ` 
  を繰り返し使うのが最も明快です。

  ```lean

   := by
      -- Finset.sum_sdiff を使って {0, 1} 
  を分離し、それらの和が0であることを示す
      ...

    -- ∑_{k=2 to n} (1/(k-1)! - 1/k!)
    -- これは Mathlib の `Finset.sum_sub_adjacent_shift` 
  のような補題で直接計算できる
    -- もしくは、帰納法で示すのが確実
  ```
