# busy period tail解

この解では、容量`1`を使い切るまでのbusy periodの尾和として期待値を読む。

独立一様乱数`U_1,U_2,...`について

```text
S_n = U_1 + ... + U_n,
tau = min { n >= 1 : S_n > 1 }
```

とする。サーバが初期仕事量`1`を持ち、各段で仕事量`U_i`を消費すると考える。`n`段後もbusy periodが終わっていない条件は

```text
S_n<=1.
```

したがって整数値変数の尾和から

```text
E[tau]=sum_{n>=0} P(S_n<=1).
```

固定した`n`では、`S_n<=1`の領域は

```text
{u_i>=0, u_1+...+u_n<=1}
```

という標準単体で、その体積は

```text
1/n!
```

である。よって

```text
E[tau]=sum_{n>=0} 1/n! = e.
```

busy periodの言葉では、サービスが続いている全prefixの体積を足しているだけで、終了時のovershootは期待値の計算に入らない。
