# 残容量Green関数解

この解では、和`S_n`ではなく、境界までの残容量を動く状態として見る。

独立な一様乱数を`U_1,U_2,...`とし、

```text
S_n = U_1 + ... + U_n,
tau = min { n >= 1 : S_n > 1 }
```

と置く。残容量を

```text
R_n = 1 - S_n
```

と書く。`R_n >= 0`である間だけ生存している。

停止時刻は、生存している状態の個数だから

```text
tau = sum_{n>=0} 1_{R_n >= 0}.
```

求めるのはこの期待値である。

## Green関数を置く

残容量が`r>=0`の状態から始めたとき、停止までに支払う単位費用の期待総量を

```text
G(r)
```

とする。元の問題は`G(1)`である。

状態`r`にいるなら、まず現在の状態で費用`1`を支払う。次の一口`U`について、

- `U > r`なら停止して以後の費用はない。
- `0 <= U <= r`なら残容量は`r-U`になり、そこから同じ問題が続く。

一様分布の密度は`1`なので、

```text
G(r) = 1 + int_0^r G(r-u) du.        (1)
```

これは停止過程のGreen関数方程式である。

## Green方程式を解く

変数を入れ替えると

```text
G(r) = 1 + int_0^r G(v) dv.
```

したがって

```text
G'(r) = G(r),
G(0) = 1.
```

よって

```text
G(r) = exp(r).
```

元の初期残容量は`1`なので、

```text
E[tau] = G(1) = exp(1) = e.
```

## この見方の芯

この証明は、`P(S_n<=1)`を個別に求めない。停止時刻を「残容量過程が殺されるまでに単位費用を何回払うか」と見て、そのGreen関数を直接解く。

一様性は、残容量`r`から次の残容量`r-U`へ移る核が区間`[0,r]`上で密度`1`になる、という一点に集約される。そのためGreen関数が自分自身の積分になり、指数関数が現れる。
