  ### **処方箋1：無限和の分解には `tsum_subtype` を使う**

  AIは `(∑' k, if k > n then ...)` 
  という条件付き無限和の扱いに最も苦労しています。これを解決する最短の道は、全単射（bijection）を手動で構
  築することでは**ありません**。より抽象的な「部分型（subtype）」の概念を利用します。

  **最重要API**: `tsum_subtype_add_tsum_subtype_compl`

  これは `api-library.md` にも記載されている、まさにこの問題のために存在する定理です。

  ```lean
  -- Summable f → (∑' (x : {x // p x}), f x) + (∑' (x : {x // ¬p x}), f x) = ∑' x, f x
  -- 意味：ある条件pを満たす元の和と、満たさない元の和を足すと、全体の和に等しい。
  ```

  これを利用した証明の骨格は以下のようになります。

  ```lean
  theorem tail_probability_formula (n : ℕ) :
      (∑' k, if k > n then hitting_time_pmf k else 0) = 1 / n.factorial := by

    -- ステップ1: if文を、部分型{k | k > n}上の和に変換する
    -- これには tsum_eq_tsum_of_ne_zero を使うのが定石
    have h_tail_eq_subtype :
        (∑' k, if k > n then hitting_time_pmf k else 0) = ∑' (k : {k // k > n}), hitting_time_pmf k := by
      -- tsum_eq_tsum_of_ne_zero を使い、k > n でない項が0になることを示す
      apply tsum_eq_tsum_of_ne_zero
      intro k
      split_ifs with h_gt
      ·-- k > n の場合、値が0でないことを示す（実際には不要）
        simp
      ·-- k ≤ n の場合、値が0であることを示す
        simp

    rw [h_tail_eq_subtype]

    -- ステップ2: 補集合の和を使って、全体の和から有限和を引く形に変形する
    -- (∑' k ∈ {k>n}) = (∑' k) - (∑' k ∈ {k≤n})
    have h_decomp := (tsum_subtype_add_tsum_subtype_compl pmf_summable ({k | k > n})).symm
    rw [h_decomp, pmf_sum_eq_one]

    -- ステップ3: {k | k ≤ n}上の無限和は、有限和と等しいことを示す
    -- これには tsum_eq_sum を使う
    have h_finite_sum : ∑' (k : {k // k ≤ n}), hitting_time_pmf k = ∑ k in Finset.range (n + 1), 
  hitting_time_pmf k := by
      -- {k | k ≤ n} が有限集合 Finset.range (n + 1) と対応することを示す
      apply tsum_eq_sum
      intro k h_mem
      -- k が n+1 以上だと hitting_time_pmf が 0 になることを示し、無限の尾が 0 であることを証明する
      -- (この部分は少し技巧的だが、sum_bijより遥かに単純)
      sorry -- ここは少し証明が必要だが、全体の構造は見通しが良くなる

    rw [h_finite_sum]
    -- あとは、以前のAIの証明でも成功していた有限和の計算に進む
    -- ...
  ```

  このアプローチの最大の利点は、`sum_bij` のような複雑な全単射の証明を、より抽象的で強力な補題に置き換える
  ことで、証明の見通しを劇的に改善する点にあります。

  ### **処方箋2：有限和の計算は `Finset.sum_sdiff` で分割する**

  AIは `∑_{k=2 to n}` の部分を計算するために `sum_bij` 
  を使って泥沼にはまっています。エラーログの多く（`introN` の失敗、`omega` 
  の失敗）はこれが原因です。これも、もっと簡単なAPIがあります。

  **推奨API**: `Finset.sum_sdiff`

  和の範囲 `Finset.range (n + 1)` を、値が `0` になる部分 (`Finset.range 2`) と、そうでない部分 
  (`Finset.range (n + 1) \ Finset.range 2`) に分割します。

  ```lean
  -- n ≥ 2 のケース
  have h_sum_finite : ∑ k in Finset.range (n + 1), hitting_time_pmf k = 1 - 1 / n.factorial := by
    -- ゴール: ∑_{k=0 to n} pmf k = 1 - 1/n!

    -- ステップ1: 和を k=0,1 の部分と k≥2 の部分に分割する
    have h_sub : Finset.range 2 ⊆ Finset.range (n + 1) := Finset.range_mono (by omega)
    rw [← Finset.sum_sdiff h_sub]

    -- ステップ2: k=0,1 の和は0なので消す
    -- `prob_tau_eq_zero_one` を使って、この部分の和が0になることを示す
    have h_zero_sum : ∑ k in Finset.range 2, hitting_time_pmf k = 0 := by
      simp [Finset.sum_range_succ, prob_tau_eq_zero_one]
    rw [h_zero_sum, add_zero]
    -- これでゴールの左辺は `∑ k in Finset.range (n + 1) \ Finset.range 2, ...` となる

    -- ステップ3: 残った和 `∑_{k=2 to n}` をテレスコープ和の公式で計算する
    -- この部分は `pmf_telescoping` を使い、帰納法や専用API `Finset.sum_range_sub_adjacent` で証明する
    -- この部分はAIの試行錯誤の中でも成功に近い部分なので、再利用できる
    -- ...
  ```

  この方法なら、複雑な添え字の全単射を証明する必要は一切なく、集合演算と算術だけで証明が完了します。

  ### **【修正版】処方箋1：`if-then-else`から`subtype`への変換**

  **目標:** `(∑' k, if k > n then f k else 0)` を `∑' (k : {k // k > n}), f k` に変換する。

  専門家が提案した`tsum_eq_tsum_of_ne_zero`は存在しませんでしたが、この変換はMathlibの標準的な手順で実現で
  きます。キーとなるのは`Set.indicator`と`tsum_subtype`です。

  **実際に使用するAPI:**
  1.  `tsum_congr`: 和の中身を同値な式に書き換える。
  2.  `Set.indicator`: `if-then-else`を表現する標準的な方法。
  3.  `tsum_subtype`: `indicator`を使った和を`subtype`上の和に変換する決定的なAPI。

  -- 専門家が "ステップ1" と呼んだ部分の、実在するAPIによる実装

  have h_tail_eq_subtype :
      (∑' k, if k > n then hitting_time_pmf k else 0) = ∑' (k : {k // k > n}), hitting_time_pmf k := by

    -- まず、if文をSet.indicatorに書き換える
    have h_if_eq_indicator : (fun k => if k > n then hitting_time_pmf k else 0) =

  ### **【修正版】処方箋2：有限和の加法的分解**

  **目標:** `∑ k in s, f k` を `(∑ k in t, f k) + (∑ k in s \ t, f k)` の形に分解する（ただし `t ⊆ s`）。

  `Finset.sum_sdiff` は `(∑ s) - (∑ t)` の形を与えるため、加法的な分解には直接使えません。しかし、集合の和
  集合(`union`)と組み合わせることで、専門家の意図を完璧に実現できます。

  **実際に使用するAPI:**
  1.  `Finset.union_sdiff_of_subset`: `s = t ∪ (s \ t)` という集合の等式を証明する。
  2.  `Finset.sum_union`: `disjoint`な集合の和集合上の和を、それぞれの和の足し算に分解する。
  3.  `Finset.disjoint_sdiff`: `t` と `s \ t` が互いに素であることを示す。

  -- まず、和の範囲であるFinset自体を分解する
  -- s = t ∪ (s \ t)
  have h_set_decomp : Finset.range (n + 1) = Finset.range 2 ∪ (Finset.range (n + 1) \ Finset.range 2) :=
