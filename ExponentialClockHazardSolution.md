# 指数時計hazard解

この解では、境界までの残り距離を「まだ鳴っていない指数時計の寿命」として読む。計算の主役は各`S_n`の体積ではなく、残り距離を持つprefixが作る時計の平均個数である。

独立一様乱数`U_1,U_2,...`について

```text
S_n = U_1 + ... + U_n,
tau = min { n >= 1 : S_n > 1 }.
```

停止時刻は、境界`1`以下にあるprefixの個数である。

```text
tau = sum_{n >= 0} 1_{S_n <= 1}.
```

したがって

```text
E[tau] = sum_{n >= 0} P(S_n <= 1).
```

## 残り距離ごとの時計

境界を`x`まで動かし、

```text
M(x) = E[ sum_{n >= 0} 1_{S_n <= x} ]
```

と置く。求めたいのは`M(1)`である。

境界を`x`まで見た時点で生きている各prefixは、残り距離`x-s`を持つ。境界をさらに`dx`だけ進めると、このprefixは次の一様増分によって新しいprefixを作るかどうかを試される。

prefixが位置`s<=x`にいるとき、

```text
s+U in (x,x+dx]
```

となる確率は、一様密度が`1`なので`dx`である。これは、境界方向に進む時計で見れば「各prefixが単位hazardで子prefixを生む」ことを意味する。

よって境界を`dx`だけ進めたときの新規prefixの期待数は

```text
M(x) dx.
```

つまり、境界方向に見たprefix集団は、現在の期待個数に比例する単位hazardを持つ。

```text
M'(x) = M(x).
```

初期条件は、空prefixが`0`に1個あるので

```text
M(0)=1.
```

したがって

```text
M(x)=exp(x).
```

## 指数時計としての読み

この微分方程式は、単位率の指数時計の生存関数を逆向きに見たものでもある。

各prefixは境界が右へ進むにつれて、幅`dx`ごとに次のprefixを生む機会を`dx`だけ持つ。したがって境界方向の「出生hazard」は常に`1`で、期待個数は指数的に増える。

境界`1`までに存在するprefixの期待個数は

```text
M(1)=exp(1)=e.
```

ゆえに

```text
E[tau]=e.
```

## この見方の芯

この証明は、`P(S_n<=1)`や`P(tau=n)`を直接計算しない。境界を進める空間方向に沿って、各既存prefixが一様密度`1`で次のprefixを薄層へ送る、というhazardだけを見る。

答えの`e`は、単位hazardで自己増殖するprefix集団の平均個数として現れる。
