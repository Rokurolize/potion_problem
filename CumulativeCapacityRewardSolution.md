# 累積容量報酬解

この解では、しきい値`1`を固定してから深さごとに数えない。残り容量`x`から始めたとき、停止前に得られる報酬の期待値を直接求める。

独立な一様乱数`U_1,U_2,...`について

```text
S_n = U_1 + ... + U_n,
tau = min { n >= 1 : S_n > 1 }.
```

残り容量が`x`の問題を考え、

```text
V(x) = E[sum_{n>=0} 1_{U_1+...+U_n <= x}]
```

と置く。求めたい値は`V(1)`である。

## 先に報酬を1つ受け取る

残り容量が`x`なら、現在のprefix自身は停止前なので、まず報酬`1`を受け取る。

次の一口`u`が`x`を超えるとそこで終わる。`0<=u<=x`なら、残り容量は`x-u`となり、同じ問題が続く。したがって、`0<=x<=1`では

```text
V(x) = 1 + int_0^x V(x-u) du.
```

変数を`t=x-u`に替えると

```text
V(x) = 1 + int_0^x V(t) dt.        (1)
```

これは停止時刻の分布を求める式ではない。停止前prefixを1個の報酬として先に取り、残り容量が小さくなった同じ報酬問題へ戻している。

## 方程式を解く

(1)から

```text
V'(x) = V(x),
V(0) = 1.
```

よって

```text
V(x) = exp(x).
```

元の問題では`x=1`なので

```text
E[tau] = V(1) = e.
```

## この見方の芯

この証明は、`P(S_n<=1)`や`P(tau=n)`を先に計算しない。停止時刻を「停止前prefixごとに1点もらう報酬」として見て、現在のprefixの報酬`1`と、その後に残る同型の問題へ分解する。

一様性は、次の一口が残り容量の中に落ちたときの密度が`1`である一点にだけ使われる。報酬値関数そのものが

```text
V = 1 + int V
```

に閉じるため、期待停止時刻は初期容量`1`での値`e`として出る。
